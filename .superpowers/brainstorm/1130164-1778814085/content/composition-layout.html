A. Three tabs (editor / composition / library)

Switch between three full-screen views via tabs. Each view has full screen real estate. Clear mental model; only one thing visible at a time. Inspired by Ableton Session/Arrangement separation, but with library as its own surface.

B. Everything visible (library strip + editor + composition strip)

Library is a horizontal pill strip at top, composition is a horizontal block strip at bottom, editor sits in the middle. Always see where you are in the song. Vertical density is real — works on desktop, harder on laptops.

C. Sidebar library + two-tab workspace

Library is a persistent left sidebar listing both patterns and compositions. The main workspace toggles between "edit pattern" and "arrange composition" via two inner tabs. Clear hierarchy: library is navigation, workspace is the work.

Common to all three: same data model, same step-stamping editor, same composition timeline shape (linear blocks pointing to library patterns as snapshot copies). The variation is purely how the user moves between surfaces and how aware they are of each at any moment.

Eye-economy note from your memory: info needed during practice (current beat, what's coming) belongs near the fretboard. Setup actions (renaming, organizing the library) can be further away. That favors layouts where library doesn't fight for attention while playing.
